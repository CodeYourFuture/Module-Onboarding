Purpose of a README File

A README file is a document, typically written in plain text or Markdown, that provides an overview of a project. Its purpose is to give users and contributors essential information such as the project's description, installation instructions, usage guidelines, and contribution rules. It serves as the first point of reference for anyone interacting with the project, ensuring clarity and ease of use.

Purpose of a Wireframe

A wireframe is a visual blueprint or skeletal outline of a webpage or application. Its purpose is to plan the layout and structure before detailed design and development begin. It helps stakeholders visualize the user interface and user experience, focusing on functionality and content placement without distractions from colors or graphics.

What is a Branch in Git?

A branch in Git is a lightweight movable pointer to a commit. Its purpose is to allow multiple lines of development to occur simultaneously. Developers use branches to work on features, fixes, or experiments independently from the main codebase, merging changes back when ready, which helps manage collaboration and version control efficiently.
