Understanding README, Wireframe and branching in git

README, wireframe and branch.

What is a README file

A README file provides information about a software project or a dataset.

Read More

A README file provides information about a project or a dataset. A README file can introduce and explain a software project including its purpose, functionality and its set up. It can also help new team members and contributors understand the project's goals, architecture and guidelines. In addition to these, a README file can give information on ideas and future plans for a project.

What is a wireframe

A wireframe is a visual diagram that shows how the elements of a website, app or other digital product relate to each other.

Read More

A wireframe is a visual diagram that shows how the elements of a website, app or other digital product relate to each other. The purpose of a wirefram is to help designers create a user-friendly layout and experience before adding more detail.

What is a branch in Git

A branch in Git is a seperate version of a work within a Git repository.

Read More

A branch in Git is a seperate version of a work within a Git repository. It allows you to work on different versions of your project simultaneously without affecting the main code base
